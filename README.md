## 📌 About The Project

**AI-Smart-Crosswalk-Project** is a real-time pedestrian safety monitoring system that uses computer vision to detect potentially risky behavior near crosswalks. The project was designed as a smart-city prototype for Holon, helping demonstrate how municipal infrastructure can use AI to identify hazards, alert operators, and improve pedestrian safety.

The system combines a **YOLOv8 pose/object detection engine** with a modern web dashboard:

- The Python AI service analyzes incoming crosswalk frames using **YOLOv8** and **OpenCV**.
- It identifies pedestrians, crowd density, phone distraction, posture cues, and estimated distance from the camera.
- When a dangerous situation is detected, the Node.js backend creates an alert, stores it in MongoDB or fallback memory, uploads/serves the annotated image, and emits a real-time `new_alert` event through **Socket.io**.
- The React dashboard displays monitored crosswalks, alert history, status indicators, images, maps, and operational statistics for fast decision-making.

This makes the project a strong proof-of-concept for smart crosswalk monitoring, traffic safety analytics, and AI-assisted municipal operations.

## ✨ Key Features

- 🚶 **Real-time pedestrian detection** using YOLOv8 pose estimation and OpenCV image processing.
- ⚠️ **Automatic hazard classification** for distracted walking, crowd density, close proximity, and risky posture.
- 📡 **Live alert pipeline** from Python AI engine to Node.js backend and React dashboard.
- 🧠 **Computer vision reasoning** with detected object counts, estimated distance, and human-readable alert reasons.
- 🖥️ **Modern dashboard UI** built with React, Tailwind CSS, charts, map views, watchlists, and alert details.
- 🔄 **Concurrent data handling** with REST APIs, Socket.io events, polling hooks, and MongoDB/fallback data support.
- 🖼️ **Annotated evidence images** generated by the AI engine and displayed in the dashboard.
- 🏙️ **Municipality-focused prototype** designed around urban pedestrian safety and crosswalk monitoring workflows.

## 🚀 Getting Started / Installation

### Prerequisites

Make sure you have the following installed:

- **Git**
- **Node.js 18+**
- **npm**
- **Python 3.10+**
- Optional: **MongoDB Atlas** account for persistent cloud storage
- Recommended: **Cloudinary** account for automatic AI alert image uploads

### 1. Clone The Repository

```bash
git clone https://github.com/liorma6/AI-Smart-Crosswalk-Project.git
cd AI-Smart-Crosswalk-Project
```

### 2. Configure Backend Environment

Create a `.env` file inside the `Backend` directory:

```env
PORT=3000
MONGO_URI=your_mongodb_connection_string
CLOUDINARY_CLOUD_NAME=your_cloudinary_cloud_name
CLOUDINARY_API_KEY=your_cloudinary_api_key
CLOUDINARY_API_SECRET=your_cloudinary_api_secret
ENABLE_AI_ENGINE=true
AI_ENGINE_POLL_INTERVAL=2
```

> If `MONGO_URI` is not configured or MongoDB is unavailable, the backend can still run with fallback in-memory data.

### 3. Install Python AI Dependencies

From the repository root:

```bash
cd Backend
python -m venv .venv
```

Activate the virtual environment:

```bash
# Windows
.\.venv\Scripts\activate

# macOS / Linux
source .venv/bin/activate
```

Install the AI dependencies:

```bash
pip install ultralytics opencv-python numpy
```

The YOLO model weights are included in the backend project. The AI service reads images from:

```text
Backend/ai_engine/test_images/
```

and writes analyzed results to:

```text
Backend/ai_engine/output_images/
```

### 4. Install Backend Dependencies

From the `Backend` directory:

```bash
npm install
```

Start the backend server:

```bash
npm start
```

The backend runs by default at:

```text
http://localhost:3000
```

> Local development note: the backend CORS configuration should allow the frontend origin `http://localhost:5173`. If you are testing locally and the browser blocks API calls, add that origin to the backend CORS allowlist.

### 5. Install Frontend Dependencies

Open a new terminal from the repository root:

```bash
cd Frontend
npm install
```

Create a local frontend environment file if needed:

```env
VITE_BACKEND_URL=http://localhost:3000
```

Start the dashboard:

```bash
npm run dev
```

The frontend runs by default at:

```text
http://localhost:5173
```

### 6. Optional: Seed Demo Data

To populate MongoDB with sample crosswalks and alerts:

```bash
cd Tests/BackTests
npm install
npm run seed
```

## ▶️ Usage

### Start The Full System

Run the backend first:

```bash
cd Backend
npm start
```

The Node.js server starts the Python YOLO worker automatically when `ENABLE_AI_ENGINE=true`.

Then run the frontend:

```bash
cd Frontend
npm run dev
```

Open the dashboard:

```text
http://localhost:5173
```

### Feed Images Into The AI Engine

For the current prototype flow, add crosswalk frames or test images to:

```text
Backend/ai_engine/test_images/
```

The AI engine monitors this directory, analyzes new image files, generates annotated output, and sends structured detection results to the backend. If a hazard is detected, a new alert appears in the dashboard in real time.

### Run The AI Service Directly

You can also run the YOLO worker manually for local testing:

```bash
cd Backend
python ai_engine/yolo_service.py
```

## 🔌 Main API Routes

| Method | Endpoint | Description |
| --- | --- | --- |
| `GET` | `/` | Backend health check |
| `GET` | `/crosswalks` | Fetch all monitored crosswalks |
| `POST` | `/crosswalks` | Create a new crosswalk |
| `GET` | `/crosswalks/:id` | Fetch a specific crosswalk |
| `GET` | `/alerts` | Fetch all alerts |
| `GET` | `/alerts/crosswalk/:id` | Fetch alerts for one crosswalk |
| `POST` | `/ai/alerts` | Create an AI-generated alert |
| `GET` | `/keep-alive` | Keep-alive endpoint for hosted deployments |

## 🧪 Development Notes

- The backend uses `socket.io` to emit `new_alert` events when the AI engine detects a dangerous situation.
- The frontend also includes polling-based update hooks for resilient dashboard refreshes.
- Annotated images are served locally from `/output_images` during development.
- In production-oriented flows, alert images can be uploaded to Cloudinary and stored as external URLs.
- Use `ENABLE_AI_ENGINE=false` when you want to run the backend API without launching the Python worker.
